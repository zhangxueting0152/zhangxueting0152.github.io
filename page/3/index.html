记录构造函数、实例、原型这些概念之间的关系

构造函数

这里的Person就是一个构造函数，里面的this指向将要生成的实例对象。每一个实例都会有自己的name,age属性，都会有自己的getInfo方法。但是有时候希望不同的实例之间共享一些属性和方法，于是产生了prototype这个东西。

原型对象

可以看到，getType方法是写在Person的prototype对象中的，这个就是原型对象。我们可以在原型对象中存放希望让所有实例共享的东西。
那Person构造函数是如何与这个原型对象关联起来的呢？其实任何一个函数（包括构造函数），都有一个prototype属性，该属性指向该函数的原型对象。这个原型对象中可以有各种属性和方法。

调试代码的时候，发现一个知识点，稍微记录一下：

proto

这个时候打开控制台检测，会发现：

原来每一个构造函数实例化的对象，都有一个proto属性，这个属性指向对应构造函数的原型对象。有些地方也把proto写成[[prototype]]。

constructor

constructor是原型对象的一个属性。

总结一下就是：prototype是构造函数的属性，它指向构造函数的原型对象。这个原型对象里的所有东西被所有的实例共享。特别的，该原型对象里有一个constructor属性，指向该构造函数。
proto是每个实例对象的具有属性，它指向上文提到的原型对象。
实例的属性由两部分组成，一个是复制构造函数的属性，每个实例的属性是互相不干扰的；另一个是所有实例共享的原型对象里的属性。

new一个对象时发生了什么：

var person = new Person();

1.var person = {};//首先创建了一个空对象person
2.person.proto = Person.prototype //将这个空对象的proto成员指向了Person函数的prototype成员对象。
3.将Perso对象里的this指针换成了person，然后调用Person函数
4.返回该对象。

使用hexo搭建博客

首先说一下什么是Hexo：
Hexo是一个快速/简洁/高效的博客框架，使用Markdown作为渲染引擎解析文章，结合丰富的主题，分分钟生成绚丽的静态网页.

准备相应的环境，包括node.js和github

github会提供帮助生成博客的方法，而hexo则会为我们的博客加上各种样式。

安装node.js和配置环境

傻瓜式操作，在命令行里输入node -v判断安装成功。

特别的，网上有些教程会建议自己修改一下npm安装的全局模块所在的路径以及缓存所在的路径，会进行其他的环境变量的配置，可以参考链接：https://www.cnblogs.com/zhouyu2017/p/6485265.html
在搜资料的过程中，发现自己一直不明白在配置环境变量的时候，什么时候使用用户变量，什么时候使用系统变量，在这里记录一下：操作系统根据Path环境变量查找命令，如果能找到，则执行该命令，否则会出现“xxx不是内部或外部命令，也不是可运行的程序或批处理文件”。而操作系统则根据PATH环境变量来查找命令。用户变量只对当前用户有效，而系统变量对所有用户有效。为减少自己所做的修改对其他人有影响，所以设置用户变量避免影响其他人。一般系统变量优先于用户变量。

安装和配置git环境

注册github账户和新建项目，博客仓库名称要遵守账户名.github.io

个人理解：就是建普通仓库的时候，其实仓库名称xxx随便起就好了，然后如果想让它发布到网上，网址就会是https://zhangxueting0152.github.io/xxx；
但是github为写博客做了专门的规定，建立博客仓库的时候，名称要遵守账户名.github.io的形式（我的账户名就是zhangxueting0152），这样github会自动生成一个我的子域名，帮助把代码转成博客，网址就是https://zhangxueting0152.github.io/

安装Hexo

在合适的位置新建文件夹，用于存放安装的东西。进入到文件夹里，

-g表示global，全局安装
开始安装。然后输入hexo –v检查是否安装成功。

等待知道出现start blogging with Hexo!

安装所需的组建

生成静态文件(g表示generate)

开启服务器(s表示service,默认打开的端口是4000)

新建一篇博客，cmd里输入hexo new post “博客名“,会发现新建的博客位于blog/source/posts文件夹下

生成以及部署文章之前，需要安装一个扩展

使用编辑器编好文章，就可以生成以及部署了。可以使用markdown，也可以使用一些在线的markdown编辑器，比如csdn提供的http://write.blog.csdn.net/mdeditor
使用编辑器编辑完博客后，你博客站点文件夹下右键空白处，选择Git Bash Here，
输入：hexo generate，回车，生成静态页面，

再输入：hexo server，回车，访问http://localhost:4000/，即可进行本地预览，

最后输入：hexo deploy，回车，同步到github上去就行了。 访问https://zhangxueting0152.github.io/
即可
